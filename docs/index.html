⚽ Anti-censura LaLiga

hay-futbol

Monitor de bloqueos de LaLiga que notifica en Mattermost cuando Internet vuelve a ser libre.

★ Ver en GitHub hayahora.futbol ↗

Qué hace

Cada vez que se ejecuta (típicamente desde cron cada 2-5 minutos), consulta la API de hayahora.futbol, determina si LaLiga tiene activo el bloqueo de IPs y envía una notificación a Mattermost solo cuando el estado cambia. Así no hay spam: un mensaje cuando empieza el bloqueo y otro cuando termina.

🚨 Bloqueo activo

¡HAY FÚTBOL! LaLiga está censurando Internet
Se han detectado bloqueos activos en las IPs monitorizadas. Si no puedes acceder a alguna web, prueba con una VPN.

✅ Internet libre

Se acabó el fútbol — Internet libre de nuevo
Los bloqueos de LaLiga han cesado. Puedes navegar tranquilamente.

Cómo funciona

Consulta la API Hace un GET a https://hayahora.futbol/estado/data.json con timeout configurable.
Calcula la proporción por ISP Para cada ISP, cuenta el % de IPs cuyo último stateChange tiene state == true.
Aplica el umbral Si algún ISP supera el BLOCK_THRESHOLD_PCT (50% por defecto), se considera bloqueo activo. Las IPs residuales sueltas no disparan alarmas.
Detecta cambio de estado Compara el estado actual con el guardado en .hayahora_last_state. Si es igual, termina sin notificar.
Notifica en Mattermost Solo si hay cambio, publica el mensaje con el detalle de ISPs afectados y actualiza el fichero de estado.
Cuando LaLiga activa el bloqueo, >90% de las IPs de cada ISP cambian simultáneamente. El umbral del 50% filtra el ruido residual de IPs que permanecen bloqueadas entre partidos.

Instalación rápida

Requisitos

bash curl jq

Configuración

# 1. Clona el repositorio
git clone https://github.com/parrazam/hay-futbol.git
cd hay-futbol

# 2. Crea tu .env a partir de la plantilla
cp env.example .env
# Edita .env con los datos de tu Mattermost

# 3. Prueba que funciona
bash hayahora_monitor.sh

Cron (cada 3 minutos)

*/3 * * * * /ruta/a/hayahora_monitor.sh >> /var/log/hayahora.log 2>&1

Configuración

Variables requeridas

VariableDescripción
MATTERMOST_URLURL base de tu instancia de Mattermost (sin / final)
MATTERMOST_TOKENToken de bot o token personal con permisos para publicar
MATTERMOST_CHANNEL_IDID interno del canal donde publicar (no el nombre)

Variables opcionales

VariableDefaultDescripción
API_URLhayahora.futbol/…URL de la API de estado
CURL_TIMEOUT15Timeout de curl en segundos
BLOCK_THRESHOLD_PCT50% mínimo de IPs bloqueadas en un ISP para considerar bloqueo real
UPTIME_KUMA_PUSH_URL—Heartbeat a Uptime Kuma al finalizar con éxito. Entrecomilla el valor si la URL lleva &.
NTFY_URL—Notificación a Ntfy cuando el script falla (indica el paso y motivo)
NTFY_TOKEN—Token Bearer para Ntfy si el topic está protegido

Observabilidad

Uptime Kuma

Si defines UPTIME_KUMA_PUSH_URL, el script hace un GET a esa URL al terminar con éxito — tanto si hubo cambio de estado como si no. Así Kuma recibe el heartbeat en cada ejecución y puede alertarte si el script deja de correr.

Ntfy

Si defines NTFY_URL, cualquier fallo inesperado del script dispara una notificación con el paso en el que falló (CURRENT_STEP), el motivo, la línea y el comando. Opcional: protege tu topic con NTFY_TOKEN.

Tests

El proyecto incluye tests de integración que levantan servidores HTTP mock (API + Mattermost) con Python y verifican las transiciones de estado. No necesitan credenciales reales. 

bash test_hayahora_monitor.sh
