¿Hay fútbol?
Detéctalo y avisa.

Script bash que monitoriza los bloqueos de LaLiga via la API de hayahora.futbol y publica en Mattermost cuando empieza o termina la censura.

Notificaciones

Avisa solo cuando importa

Cuando empieza el bloqueo, recibes un aviso. Cuando termina, otro. Sin spam: el script compara con el último estado guardado y solo publica si cambió.

🚨
Bloqueo activo

¡HAY FÚTBOL! LaLiga está censurando

Se han detectado bloqueos activos en las IPs monitorizadas. El mensaje incluye el desglose por ISP afectado. Si no puedes acceder a alguna web, prueba con una VPN.

✅
Internet libre

Se acabó el fútbol — Internet libre

Los bloqueos de LaLiga han cesado. La proporción de IPs bloqueadas ha caído por debajo del umbral. Puedes navegar tranquilamente.

Cómo funciona

Detección por proporción

Cuando LaLiga activa el bloqueo, más del 90% de las IPs de cada ISP cambian a la vez. El script aprovecha ese patrón en lugar de contar IPs sueltas.

Consulta la API

GET a hayahora.futbol/estado/data.json con timeout configurable.

Calcula la proporción por ISP

Para cada ISP, cuenta el % de IPs cuyo último stateChange tiene state == true.

Aplica el umbral

Si algún ISP supera BLOCK_THRESHOLD_PCT (50% por defecto), se considera bloqueo activo. Las IPs residuales sueltas no disparan alarmas.

Detecta cambio de estado

Compara el estado actual con el guardado en .hayahora_last_state. Si es igual, termina sin notificar.

Notifica en Mattermost

Solo si hay cambio, publica el mensaje con el detalle de ISPs afectados y actualiza el fichero de estado.

💡
Con el umbral del 50%, las IPs sueltas que quedan en true entre partidos se filtran como ruido. Solo se considera bloqueo cuando un ISP entero pasa al modo censura.
Instalación

En menos de un minuto

Solo necesitas bash, curl y jq. Disponibles en cualquier distro Linux moderna.

Requisitos

bash curl jq

Configuración

bash
# 1. Clona el repositorio
git clone https://github.com/parrazam/monitor-hay-futbol.git
cd monitor-hay-futbol

# 2. Crea tu .env a partir de la plantilla
cp env.example .env
# Edita .env con los datos de tu Mattermost

# 3. Prueba que funciona
bash hayahora_monitor.sh

Programar con cron (cada 3 minutos)

crontab
*/3 * * * * /ruta/a/hayahora_monitor.sh >> /var/log/hayahora.log 2>&1
Configuración

Variables del .env

Tres variables obligatorias para Mattermost y un puñado de opcionales para ajustar el comportamiento y la observabilidad.

Requeridas

MATTERMOST_URLURL base de tu instancia (sin / final)

VariableDefaultDescripción

Observabilidad

¿Y si el script falla?

Dos integraciones opcionales para saber siempre si el monitor está vivo y por qué se cayó si se cae.

Uptime Kuma

Si defines UPTIME_KUMA_PUSH_URL, el script hace un GET a esa URL al terminar con éxito — tanto si hubo cambio como si no. Kuma recibe heartbeat en cada ejecución y puede alertarte si el cron deja de correr.

Ntfy

Si defines NTFY_URL, cualquier fallo inesperado dispara una notificación con el paso (CURRENT_STEP), el motivo, la línea y el comando. Protege el topic con NTFY_TOKEN si quieres.
